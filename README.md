# 事務ツール

請求書・領収書・見積書・支払明細書・納品書をローカルで作成・PDF出力できる
デスクトップアプリ（Windows / Mac / Linux 対応）。

## できること

- 書類作成：請求書 / 領収書 / 見積書 / 支払明細書 / 納品書
- PDF 出力（日本語対応、A4、@page 設定済み）
- 会社マスタ / 取引先マスタ / 品目マスタ
- 角印の合成（画像アップロード + 位置・サイズ指定）
- 取引先ごとの書類番号プレフィックス（全社共通連番 / 取引先別連番 / 履歴から自動追従）
- データ保存先のカスタマイズ（OneDrive 等との同期可能）
- 書類履歴の検索・複製

---

## 技術スタック

- Electron 30 + Next.js 14（Pages Router） + TypeScript 5
- better-sqlite3 + Drizzle ORM（ローカル SQLite、将来 Postgres 移行可）
- Tailwind CSS + shadcn/ui
- React Hook Form + Zod + TanStack Query + Zustand
- PDF 生成：Electron `webContents.printToPDF()`
- パッケージング：electron-builder（Windows / Mac / Linux 対応）

## 動かしてみる

### 前提
- Node.js 20 以上
- git
- Python 3 と C++ ビルドツール（`better-sqlite3` のネイティブビルドに必要）

### 手順

```bash
git clone https://github.com/Aizoon-jp/document-tool.git
cd document-tool
npm install
npm run dev
```

Electron のウィンドウが立ち上がれば成功です。

## 詰まりそうなポイント（先出し）

このツールの作成で実際にハマりどころだったところです。
Claude Code 単体で挑戦する場合、以下で手が止まる可能性が高いです。

- **better-sqlite3 のネイティブビルド** — Python / ビルドツールの準備、Electron バージョンとの整合
- **Nextron（Next.js + Electron）の環境構築** — ポート設定、preload、`contextIsolation`、`nodeIntegration`
- **IPC 通信の型共有** — main ↔ renderer のメッセージ型を安全に受け渡す設計
- **PDF 生成** — 非表示 BrowserWindow、`@page` ルール、`mm` 単位レイアウト、日本語フォント同梱
- **electron-builder** — `appId` / `productName` / 署名 / リソースパス / ネイティブモジュール同梱
- **マイグレーション運用** — 開発時と本番ビルド後でマイグレーションファイルのパスが変わる罠
- **GitHub Actions でのクロスビルド** — Windows / Mac / Linux を並列で回す設定

動いている完成品が手元にあるのに、同じ構成で一から作り直すのは思ったより難しい、
というのが実験の肝です。
