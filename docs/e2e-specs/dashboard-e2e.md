# P-001 ダッシュボード E2Eテスト仕様書

生成日: 2026-04-20
対象ページ: `/`
ページコンポーネント: `renderer/pages/index.tsx`
レイアウト: `renderer/layouts/MainLayout.tsx`
権限レベル: なし（ローカル版は認証不要 / 単一ユーザー）

---

## ユーザーゴール

1. 最近発行した書類を一目で確認できる
2. 書類種別を選んで新規書類作成を即座に開始できる
3. 今月の発行状況（総件数 / 種別内訳）を把握できる
4. 過去書類にワンクリックで到達できる（詳細 / 履歴一覧）

---

## テスト環境

```yaml
URL: http://localhost:3055/  （開発時Next.js devサーバー）
起動: npm run dev （Electron + Next.js）
認証: 不要（ローカル単一ユーザー）
DB: better-sqlite3（app.getPath('userData')/data.db）
前提: documentsテーブルに5件のサンプルデータが投入済みであること
      （ゼロ件テストは専用シナリオで別DBスナップショットを使用）
```

### 前提条件

- Phase 3完了時点ではIPC未接続のため、UI内のハードコード定数（`RECENT_DOCUMENTS`, `MONTHLY_SUMMARY`）でPASSすること。
- Phase 8のAPI統合後は、同一スキーマ（`Document`, `MonthlySummary` 型）を返すIPCに差し替えて同じテストがPASSすることを確認する。

## テストケース詳細

### TC E2E-DASH-001: ダッシュボード初期表示

- **目的**: ユーザーが `/` にアクセスした直後、ページの全セクションが欠けなく表示されることを保証
- **事前状態**: アプリ起動直後、`/` にアクセス
- **手順**:
  1. `page.goto('/')`
  2. `page.waitForLoadState('networkidle')`
- **期待結果**:
  - `page.getByRole('heading', { level: 1, name: 'ダッシュボード' })` が可視
  - 説明文「最近の書類と、書類種別ごとのクイック作成が利用できます。」が表示
  - クイック作成セクション見出し「クイック作成」が表示
  - クイック作成ボタンが5個描画される（`button` 要素、テキスト: 請求書/領収書/見積書/振込依頼書/納品書）
  - 「最近発行した書類」カード（見出し + 説明「直近5件を表示」）が表示
  - 「今月の発行件数」カードが表示（現在年月ラベル `yyyy年M月` を含む）
  - 「書類履歴をすべて見る」ボタンが表示

---

### TC E2E-DASH-002: クイック作成フロー（全5種）

- **目的**: 5種類の書類タイプ全てで書類作成ページへ正しく遷移すること
- **事前状態**: `/` 表示中
- **手順 & 期待結果**（パラメータ化テスト推奨）:

| 入力ボタンテキスト | クリック後URL |
|------------------|--------------|
| 請求書 | `/documents/new?type=invoice` |
| 領収書 | `/documents/new?type=receipt` |
| 見積書 | `/documents/new?type=quote` |
| 振込依頼書 | `/documents/new?type=payment_request` |
| 納品書 | `/documents/new?type=delivery_note` |

- **Playwright例**:
  ```ts
  await page.getByRole('button', { name: '請求書' }).click()
  await expect(page).toHaveURL(/\/documents\/new\?type=invoice$/)
  ```

---

### TC E2E-DASH-003: 最近書類の行クリックで詳細遷移

- **目的**: 最近発行テーブルの行クリックで個別詳細ページへ遷移すること
- **事前状態**: `/` 表示中、最近書類5件がレンダリング済
- **手順**:
  1. 1行目（`tbody > tr` の1番目）をクリック
- **期待結果**:
  - URLが `/documents/1` に遷移する
  - 行にカーソルポインタスタイル（`cursor-pointer`）が適用されている（任意検証）

---

### TC E2E-DASH-004: 最近書類のフォーマット検証

- **目的**: 日付・金額・種別ラベル・書類番号が仕様どおりフォーマットされていること
- **事前状態**: `/` 表示中
- **手順 & 期待結果**:

1行目セル内容の期待値（UIハードコード値に基づく）:

| 列 | セレクタ目安 | 期待値 |
|----|-------------|--------|
| 発行日 | `tbody tr:nth-child(1) td:nth-child(1)` | `2026/04/18` |
| 書類種別 | `tbody tr:nth-child(1) td:nth-child(2)` | `請求書` |
| 取引先 | `tbody tr:nth-child(1) td:nth-child(3)` | `株式会社サンプル` |
| 書類番号 | `tbody tr:nth-child(1) td:nth-child(4)` | `2026-04-003` |
| 金額 | `tbody tr:nth-child(1) td:nth-child(5)` | `￥1,320,000` （Intl.NumberFormat ja-JP JPY） |

- **補足**: 5行すべてが `DOCUMENT_TYPE_LABEL` にマップされた日本語で表示されること。

---

### TC E2E-DASH-005: 今月サマリ表示

- **目的**: 月次発行件数サマリの総件数と種別内訳が正しく表示されること
- **事前状態**: `/` 表示中
- **手順**:
  1. 「今月の発行件数」カード内のテキストを取得
- **期待結果**:
  - ヘッダーに現在年月（例: `2026年4月`）が表示（`format(new Date(), 'yyyy年M月')`）
  - 総件数 `12` が大きく表示
  - 補足テキスト「件発行」が表示
  - 内訳が5行表示される:
    - 請求書: `5件`
    - 領収書: `3件`
    - 見積書: `2件`
    - 振込依頼書: `1件`
    - 納品書: `1件`

---

- **目的**: フッター右下のボタンから書類履歴一覧に遷移できること
- **事前状態**: `/` 表示中
- **手順**:
  1. `page.getByRole('button', { name: '書類履歴をすべて見る' })` をクリック
- **期待結果**:
  - URLが `/documents` に遷移する

---

### TC E2E-DASH-007: 空状態（最近書類ゼロ件）

- **目的**: 書類が1件も登録されていない初回起動状態でもページが破綻しないこと
- **事前状態**: documentsテーブル空のDBスナップショットで起動
- **手順**:
  1. `page.goto('/')`
- **期待結果**:
  - ページが例外なく描画される（見出し・クイック作成ボタン5個・両カードが可視）
  - 最近発行テーブルの `tbody > tr` が0行
  - 今月サマリの総件数が `0`、内訳5種すべて `0件`
  - クイック作成ボタンは通常どおりクリック可能で `/documents/new?type=...` に遷移する

## セレクタ指針

Playwrightテスト実装時の推奨セレクタ:

| 要素 | 推奨セレクタ |
|-----|------------|
| ページ見出し | `page.getByRole('heading', { level: 1, name: 'ダッシュボード' })` |
| クイック作成ボタン | `page.getByRole('button', { name: '請求書' })` 等（書類ラベル名） |
| 最近書類テーブル | `page.locator('section').filter({ hasText: '最近発行した書類' }).locator('table')` |
| 最近書類の行 | `table tbody tr`（インデックス指定） |
| 今月サマリカード | `page.getByText('今月の発行件数').locator('..').locator('..')` |
| 履歴リンク | `page.getByRole('button', { name: '書類履歴をすべて見る' })` |

> 実装時にdata-testid追加が必要と判断したら、本仕様書を更新のうえコード側にも反映する。
