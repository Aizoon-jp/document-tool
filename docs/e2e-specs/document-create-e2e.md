# P-002 書類作成 E2Eテスト仕様書

生成日: 2026-04-20
対象ページ: `/documents/new`
ページコンポーネント: `renderer/pages/documents/new.tsx`
レイアウト: `renderer/layouts/MainLayout.tsx`
権限レベル: なし（ローカル版は認証不要 / 単一ユーザー）

---

## ユーザーゴール

1. ダッシュボードのクイック作成から書類種別を指定した状態で即フォームを開始できる
2. 書類種別・取引先・発行日・明細を最小入力で確定し、右ペインのプレビューで仕上がりを確認できる
3. オプション（消費税/軽減税率/源泉徴収/備考/振込先）を切り替えて金額計算とプレビューに反映できる
4. 印影を選択し、PDF生成/下書き保存/キャンセルのいずれかで作業を完了・中断できる

---

## テスト環境

```yaml
URL: http://localhost:3055/documents/new （開発時Next.js devサーバー）
起動: npm run dev （Electron + Next.js）
認証: 不要（ローカル単一ユーザー）
DB: 未接続（Phase 4 時点）。クライアント/品目/印影は mockData.ts のハードコード定数
前提:
  - MOCK_CLIENTS に3件（c1:株式会社サンプル / c2:有限会社テスト商事 / c3:合同会社アイゾーン）
  - MOCK_ITEMS に5件（i1〜i5。i5は軽減税率8%）
  - MOCK_STAMPS に2件（s1:角印（代表）= isDefault:true / s2:角印（営業）= isDefault:false）
```

### 前提条件

- Phase 4 時点では IPC 未接続のため、`renderer/components/documents/mockData.ts` の定数でPASSすること。
- 書類番号は `generateDocumentNumber()`（`utils.ts`）によるダミー採番で、`{PREFIX}-{YYYY}-{MM}-001` 形式。DBシーケンスが接続される Phase 8 以降は末尾連番部分のみ差し替わることを想定。
- 発行日デフォルトは `new Date().toISOString().slice(0, 10)`（実行日=システム日付）。テスト時は `page.clock.install()` 等で日付を固定することを推奨。
- PDF生成・下書き保存ボタンは現状 `alert()` + `console.log()` のダミー動作（`onSubmit`/`onSaveDraft`）。フロントの挙動のみ検証する。

## テストケース詳細

### TC E2E-DOC-NEW-001: クエリパラメータ付き初期表示

- **目的**: ダッシュボードのクイック作成から書類種別付きで来たときに、全初期値が type に応じて設定されていること
- **事前状態**: アプリ起動直後
- **手順**:
  1. `page.goto('/documents/new?type=invoice')`
  2. `page.waitForLoadState('networkidle')`
- **期待結果**:
  - `page.getByRole('heading', { level: 1, name: '書類作成：請求書' })` が可視
  - 説明文「取引先・明細・オプションを入力すると、右側にプレビューが自動で反映されます。」が表示
  - 書類種別セレクト（`#documentType`）の表示値が `請求書`
  - 取引先セレクト（`#clientId`）は placeholder `取引先を選択`
  - 発行日（`#issueDate`）に今日の ISO 日付（`yyyy-MM-dd`）がセット
  - 書類番号（`#documentNumber`）が正規表現 `^INV-\d{4}-\d{2}-001$` にマッチ
  - 明細モードは `直接記載` が選択、明細行が1行描画済
  - オプションセクションで `振込先情報を表示` の Switch が checked（その他のうち `消費税を表示` も checked）

  - 右ペイン `プレビュー` カード内の `<h2>` が `請 求 書` を含む

## セレクタ指針

Playwright/Electronテスト実装時の推奨セレクタ:

| 要素 | 推奨セレクタ |
|-----|------------|
| ページ見出し | `page.getByRole('heading', { level: 1 })`（動的に `書類作成：XXX`） |
| 書類種別セレクト | `page.locator('#documentType')` + `page.getByRole('option', { name: '領収書' })` |
| 取引先セレクト | `page.locator('#clientId')` + `page.getByRole('option', { name: /株式会社サンプル/ })` |
| 発行日 | `page.locator('#issueDate')` |
| 書類番号 | `page.locator('#documentNumber')` |
| 明細モードラジオ | `page.getByRole('radio', { name: '直接記載' })` / `'別紙明細の通り'` |
| 明細行の品目セレクト | 1行目の `role=combobox` 相当（`SelectTrigger`）をインデックス指定 |
| 明細追加ボタン | `page.getByRole('button', { name: '明細行を追加' })` |
| 明細削除ボタン | `page.getByRole('button', { name: '明細行を削除' })`（aria-label） |
| 別紙合計入力 | `page.locator('#externalAmount')` |
| オプションSwitch | `page.locator('#opt-includeTax')`, `#opt-reducedTaxRate`, `#opt-withholdingTax`, `#opt-showRemarks`, `#opt-showBankInfo` |
| 印影ボタン | `page.getByRole('button', { name: /角印（代表）/ })`, `/角印（営業）/` |
| プレビュー見出し | `page.locator('section').filter({ hasText: 'プレビュー' }).locator('h2')` |
| 右上アクション | `page.getByRole('button', { name: 'PDF生成' })` / `'下書き保存'` / `'キャンセル'` |
| 備考テキストエリア | `page.getByRole('textbox', { name: /お客様に伝える補足/ })`（Placeholder経由） |

> `#opt-*` ID は実装上 `<Switch id="opt-{key}">` として付与済み。data-testid は不要と判断。必要に応じて印影ボタンや明細行に data-testid を追加する場合は本仕様書も更新する。
